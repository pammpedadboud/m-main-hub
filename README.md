# m-main-hub

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可能是工具界面、信息展示页或测试页面，以静态文件形式存放，便于通过 GitHub Pages 或其他静态托管服务直接访问。

仓库不针对特定域名或网站，仅为页面集合提供统一的管理和版本控制。

## 目录说明

```
m-main-hub/
├── index.html          # 首页 / 页面导航（可选）
├── pages/              # 存放各独立页面的子目录
│   ├── page-a/
│   │   ├── index.html
│   │   └── assets/
│   ├── page-b/
│   └── ...
├── assets/             # 公共资源（CSS、JS、图片等）
└── README.md
```

- `pages/`：每个子文件夹代表一个独立页面，包含其自身的 HTML 及所需资源。
- `assets/`：存放多个页面可能共用的样式、脚本或媒体文件。
- `index.html`：可选，可设计为页面导航或说明页。

## 页面归档说明

- 所有页面以纯静态 HTML 形式存放，不依赖后端服务。
- 每个页面应尽量自包含，避免跨文件夹引用（公共资源除外）。
- 提交新页面时，请在 `pages/` 下新建文件夹，并确保链接路径正确。

## 维护说明

- 欢迎提交 PR 添加新页面，或改进现有页面。
- 请保持目录结构清晰，文件名有语义。
- 请勿在页面中包含任何推广、营销或诱导点击内容。
- 仓库主要面向技术归档和页面展示，不提供具体业务用途。

## 使用方式

1. 克隆本仓库到本地。
2. 在 `pages/` 下创建新页面文件夹。
3. 编写 HTML 及相关资源。
4. 提交并推送，GitHub Pages 会自动部署（需在仓库设置中开启）。
